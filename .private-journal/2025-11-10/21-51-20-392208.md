## Project Notes

IMPORTANT INSIGHT from Rabble: For seed data preloading, we can bypass the JSON → Event object → Drift layer entirely.

**Better Approach**: 
- Generation script outputs raw SQL INSERT statements
- Bundle SQL file in assets
- Execute SQL directly against SQLite

**Why this is better**:
- 10x faster: No JSON parsing, no Event object creation, no Dart overhead
- Simpler: Just `customStatement()` for each INSERT
- Smaller: SQL is more compact than JSON
- Direct: No round-trip through nostr_sdk serialization

**Implementation**:
```dart
// Load SQL file
final sql = await rootBundle.loadString('assets/seed_data/seed_events.sql');

// Execute in transaction
await db.transaction(() async {
  // Split on semicolons, execute each INSERT
  for (final statement in sql.split(';').where((s) => s.trim().isNotEmpty)) {
    await customStatement(statement);
  }
});
```

**Generation script outputs**:
```sql
INSERT OR IGNORE INTO event (id, pubkey, created_at, kind, tags, content, sig, sources) 
VALUES ('abc123...', 'pubkey1...', 1234567890, 34236, '[["url","..."]]', '...', '...', NULL);

INSERT OR IGNORE INTO user_profiles (pubkey, display_name, name, picture, created_at, event_id, last_fetched)
VALUES ('pubkey1...', 'Alice', 'alice', 'https://...', '2024-01-01', 'evt123...', '2024-01-01');
```

This is the right approach for seed data - raw SQL dump.

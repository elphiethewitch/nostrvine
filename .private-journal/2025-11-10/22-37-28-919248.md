**Implementation approach**:

- Save to `assets/seed_media/` directory:

  - `thumbnails/{event_id}.jpg`
- On first launch, copy from assets to VideoCacheManager cache directory
- VideoCacheManager uses flutter_cache_manager which stores in temp directory

**APK size impact**:

- 10 thumbnails × ~50KB = ~0.5MB
- Total: ~10-15MB additional (acceptable per Rabble's decision)

**Cache initialization flow**:
1. SeedDataPreloadService loads event SQL
2. New: SeedMediaPreloadService copies bundled media to cache
3. VideoCacheManager.initialize() sees pre-populated cache
